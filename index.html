OpenFusion

An entirely free game engine, made for performance and an alternative to Clickteam Fusion.


Why OpenFusion?

OpenFusion is multi-platform website game engine that's very light-weight. The program is designed to very light-weight and efficent, so that no package manager are needed for the program; it's also blazingly fast when you're trying to compile your script with the Dwarf compiler. The entire codebase is open-source, so everyone can use/distribute it for free.

    Getting Started

      First, you need to download the repository ZIP file up there, and then install NodeJS with this link; after you've installed NodeJS and downloaded the repository ZIP file, you may now see the OpenFusion in your download folder. After everything is done, open index.html inside the OpenFusion folder and enjoy creating your application!

    Classic "Hello, World!" Application

    If you've never heard about "Hello, World" program before, it's just a program displaying the word "Hello, World" on the screen. In OpenFusion, you'll need to use the Dwarf language and the OpenFusion editor to get the work done.

    First, click the new application on the menu, then click create; after that you'll go inside the editor. After you're in the editor, open click the script editor and paste the code below.

    MyApp.dw

    moduleFrom "./MyApp.json" importTo "./MyApp.js" importFrom $MyString as string when IS LOADED locate $MyString center center size $MyString auto auto setAttribute $MyString "font" "monospace" setAttribute $MyString "size" 32 setAttribute $MyString "text" "Hello, World!" show $MyString end

    After you've copy this exact script, you can now compile it into JavaScript's format, by using the dwc compiler; then finally, load the script and test the application. Here's what'll happen inside the application.

    Learning the Syntaxes

    Dwarf is language that's very easy to learn. In the future, the language would be implemented in C++, for faster compilation. Here's all keywords that you'll need to learn.

    keywords.dw

    # This line is a comment, the compiler will cut this part out of its way. moduleFrom [application_filename (string)] # example for moduleFrom: moduleFrom "./MyApp.json" exportTo [target_filename (string)] # example for exportTo: exportTo "./MyApp.js" importFrom [instance_name (string)] # example for importFrom: importFrom $MyString as string when [logical_operator (logic)] [condition (symbol)/[instance_id (instance)] [action_name (action)] [...] # example for when: when IS $MyString HOVER # when IS $MyString HOVER AND NOT $MyString HOVER locate [instance_id (instance)] [x_position (integer)/center] [y_position (integer)/center] # example for locate: locate $MyString center center size [instance_id (instance)] [width (integer)/full] [width (integer)/full] # example for size: size $MyString full full setAttribute [instance_id (instance)] [attribute_name (string)] [attribute_value (string)] # example for setAttribute: setAtribute $MyString "text" "Hello, World!" show [instance_id (instance)] # example for show: show $MyString show [instance_id (instance)] # example for show: show $MyString end # note: end is a keyword to end the when statement # example for end: end

    Please take your time and read all informations about the keywords.
